## On-Demand Diagnostics
For single queries use the dedicated functions: dns_lookup, ssl_check, whois_lookup, http_check,
network_check, seo_check, email_check, blacklist_check, port_scan, smtp_test, geo_check.
For a complete audit of one domain use domain_full_check (runs all checks in parallel).

## Domain Health Monitoring (3-step setup)
A "monitor" in web-tools is a domain health monitor — a saved configuration that runs DNS/SSL/HTTP/email
checks on a group of domains on demand or at a scheduled interval.
It is NOT an automation rule — never use the automations extension for domain monitoring.

Step 1: create_domain_group — define which domains to watch (max 5 groups, 20 domains each)
Step 2: create_check_profile — choose which checks to run: dns/ssl/http/email/blacklist/geo (max 5 profiles, 5 checks each)
Step 3: create_monitor — link group + profile + interval → saves a wt_monitors record (max 5 monitors)

After setup: run_scan — trigger a scan immediately; get_scan_results — read the latest snapshot.
Results appear in the panel automatically after each scan.

## Interpreting Results
- **DNS**: No MX records = email is broken. Propagation issues = possible split-brain or TTL problem.
- **SSL**: Grade F or expired = urgent. Days-until-expiry < 14 = renew now.
- **HTTP**: Grade F = missing critical security headers (HSTS, CSP, X-Frame-Options).
- **Email**: SPF missing = spoofing risk. DMARC p=none = monitoring only, not enforcing. DKIM missing = delivery issues.
- **Blacklist**: Verdict "critical" (Spamhaus ZEN/SBL) = email severely impacted. Act immediately.
- **Geo**: Mismatch across regions = possible anycast misconfiguration or split-DNS.

## Chaining Actions
After domain_full_check, highlight the most critical issues first. If a user repeatedly checks the
same domain, suggest setting up a monitor. After creating a group and profile, suggest create_monitor.
Format DNS records as tables, show SSL expiry prominently, flag blacklist issues as urgent.
